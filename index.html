May 2026 – Jun 2026 · Featured

Movie Caching Engine

A single eviction policy fails under mixed access patterns — LRU thrashes on frequency-skewed workloads, LFU on recency-skewed ones. Designed a two-tier strategy: LRU at L1 for hot recency bias, LFU at L2 as a frequency-weighted fallback. Enforced O(1) on all ops via doubly-linked list + HashMap. Applied Facade, Strategy & Chain of Responsibility to keep eviction policy swappable without touching core logic. Validated under concurrent load with JUnit.

Java LRU/LFU Design Patterns OOP JUnit

Rakuten Symphony · Production · 2023–Present

4G Load Balancing — LBIAC

When a cell becomes congested, naive offloading causes interference across neighbors. Implemented LBIAC — a 3GPP X2AP inter-eNB coordination protocol — using a 5-state FSM (L0–L4) tracking DL PRB and time-domain load independently. Each of 32 neighbor cells runs its own nested state machine through measurement request → response → periodic update, with configurable backoff timers preventing hammering unresponsive peers. Deployed for Drillisch and Rakuten Mobile production networks.

C++ Java 3GPP X2AP FSM 4G LTE Distributed Systems

Rakuten Symphony · Production · 2023–Present

X2AP Private IEs — Overload Stability

Standard 3GPP stop-reporting on overload causes signaling thrash — load oscillates around the threshold, restarting the resource status procedure repeatedly. Designed vendor-specific Private IEs extending the X2AP Resource Status Update: a configurable pause indicator (vs hard stop) with a 14,400s countdown. Source and target sides manage asymmetric timer state — pause on sender, backoff-aware countdown on receiver — with graceful fallback for peers that only support standard 3GPP. Eliminated oscillation without sacrificing recovery responsiveness.

C++ X2AP 3GPP Protocol Design State Machine Telecom
